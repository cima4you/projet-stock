# Stock Management System

## Overview

This is a bilingual (Arabic and French) inventory management system built with Flask, designed to provide comprehensive stock management capabilities for businesses. The application features user authentication, product management, stock movement tracking, automated email notifications, and detailed reporting functionality. The system supports both RTL (Arabic) and LTR (French) layouts with complete translation support.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture
- **Framework**: Flask with Jinja2 templating engine for server-side rendering
- **UI Framework**: Bootstrap 5.1.3 for responsive design and consistent styling
- **Icons**: Font Awesome 6.0.0 for comprehensive iconography
- **JavaScript**: Vanilla JavaScript for client-side interactions including form validation, table features, and component initialization
- **Styling**: Custom CSS with Bootstrap customization and bilingual design elements (RTL/LTR support)
- **Language Support**: Complete bilingual interface (Arabic RTL and French LTR) with dynamic language switching via translations.py

### Backend Architecture
- **Framework**: Flask (Python web framework) with modular route organization
- **Database**: SQLite for local data persistence with automatic schema management
- **Authentication**: Session-based authentication using Werkzeug password hashing
- **Email System**: SMTP integration via Gmail for automated notifications and alerts
- **File Processing**: Pandas integration for Excel import/export operations
- **Image Processing**: PIL (Pillow) for logo management and image handling
- **Security**: Environment variable configuration (.env) and secure file upload handling
- **Internationalization**: Built-in translation system supporting Arabic and French languages

## Key Components

### Authentication & User Management
- **Session-based Authentication**: Uses Werkzeug for secure password hashing and Flask sessions
- **Three-tier Role System**: user, admin, and principal_admin with different permission levels
- **User CRUD Operations**: Complete user management with email validation and role assignment
- **Password Security**: Secure password handling with hash generation and verification
- **Password Reset**: Email-based password reset functionality with secure token generation

### Inventory Management
- **Product Management**: Complete CRUD operations for products with codes, names, categories, units, quantities, and expiration dates
- **Stock Movement Tracking**: Comprehensive tracking of inventory entries and exits with audit trails
- **Automatic Movement Creation**: System automatically creates entry movements when adding new products
- **Low Stock Monitoring**: Dashboard alerts for products with low inventory levels
- **Expiration Tracking**: Monitor products approaching or past expiration dates

### Email Notification System
- **SMTP Configuration**: Gmail integration for sending automated notifications
- **Recipient Management**: Manage email recipients for different types of notifications
- **Email Testing**: Built-in email connection testing and test message sending
- **Environment-based Configuration**: Secure email credentials management via environment variables
- **Bilingual Email Templates**: HTML email templates in both Arabic and French for notifications

### Logo & Branding Management
- **Logo Upload**: Support for PNG, JPG, JPEG image uploads
- **Image Processing**: Automatic resizing and optimization using PIL
- **Brand Customization**: Logo display throughout the application interface

### Reporting System
- **Product Reports**: Detailed product inventory reports with filtering and export capabilities
- **Movement Reports**: Historical stock movement tracking and analysis
- **Excel Export**: Generate Excel reports using xlsxwriter
- **Advanced Filtering**: Date range, category, and type-based filtering

## Data Flow

### Authentication Flow
1. User login with username/password
2. Werkzeug password verification
3. Session creation and role-based access control
4. Automatic logout on session expiry

### Inventory Management Flow
1. Product creation with automatic entry movement generation
2. Stock level updates through movement transactions
3. Real-time inventory calculation
4. Low stock and expiration alerts

### Notification Flow
1. System events trigger email notifications
2. SMTP connection to Gmail servers
3. Bilingual template selection based on user preference
4. Automated delivery to configured recipients

## Deployment Strategy

### Local Development
- SQLite database for rapid prototyping
- Environment variables via .env file
- Local file storage for uploads and logos
- Development server via Flask's built-in server

### Production Considerations
- **Database**: Currently uses SQLite (code agent may add Postgres later for production scaling)
- **File Storage**: Local filesystem (may need cloud storage for production)
- **Email Configuration**: Gmail SMTP (production may require dedicated email service)
- **Session Management**: Uses secure secret keys from environment variables
- **Security**: Password hashing, secure file uploads, and environment-based configuration

### Configuration Management
- Environment variables for sensitive data (email credentials, session secrets)
- Configurable upload directories
- Multilingual support without code changes
- Modular template structure for easy customization

The application is designed to be easily deployable with minimal configuration changes, supporting both development and production environments through environment variable configuration.
